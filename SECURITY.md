# Security Policy

## Reporting a Vulnerability

Open a private security report through your repository hosting platform or contact maintainers directly.

Include:
- affected component
- reproduction steps
- impact assessment
- suggested mitigation

## Safe Usage

This project is for authorized security testing only.

- Do not scan targets without explicit permission.
- Configure `policy.json` scope and tool restrictions before running.
- Use API authentication (`KERAUNOS_API_KEY`) in non-local environments.
